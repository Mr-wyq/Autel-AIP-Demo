Agent生产、运行、开放与持续优化主流程 点击流程节点可进入对应页面或下方页面映射
内部系统
模型平台 / 数据平台 / 仿真平台 / 设备平台 / 无人机控制系统 / 其他内部业务系统
外部系统
客户系统 / 第三方平台 / 协作平台 / API、SDK、Webhook 调用方
↑通过 API / SDK / Webhook / 协作平台 / 内部系统集成，面向内部系统和外部系统开放业务Agent
④ 生态开放Agent服务出口
面向外部系统、内部系统与协作平台开放业务Agent开放配置 → 授权治理 → 调用运行 → 回调监控
开放配置Agent、版本、API、SDK、Webhook 授权治理应用、API Key、Scope、配额 调用运行提交任务、查询状态、获取结果 回调监控Webhook、失败重试、调用审计
↑业务Agent向生态开放提供可调用Agent能力
③ 业务Agent生产运行
承接真实任务、Session和运行记录注册 → 服务 → 执行 → 状态 → 记录 → 反馈
注册进入业务Agent目录 服务任务与Session入口 执行调用模型、数据、Tool、设备 状态运行、等待、失败、完成 记录Trace、Files、Audit 反馈评价、失败样例、优化建议
↑Agent工程发布并注册为业务Agent
② Agent工程生产Agent
把业务需求生产成可发布Agent定义 → 装配 → 运行 → 验证 → 发布 → 监控优化
定义角色、目标、Prompt、模型、边界 装配知识、Tool、Workflow、Skill、设备 运行Environment、Session、State、Event 验证Test、Debug、Trace、Eval、Simulation 发布版本、审批、灰度、回滚 监控优化质量、成本、延迟、失败原因
↑基础服务向Agent工程提供模型、数据、仿真和设备能力
① 基础服务能力供给
四个子系统向AIP-Agent开放的服务并行供给，不包含子系统自身建设功能
AIP-Agent服务目录与授权：发现可用服务、配置凭证和权限、检查健康状态；Agent装配时绑定，业务Agent运行时调用。
持续优化回路：外部调用日志、回调结果、业务Agent状态、Trace、失败样例和用户反馈 → Observability → 返回定义、装配、验证或发布环节继续迭代。

生态开放

开放配置 -> 授权治理 -> 调用运行 -> 回调监控

开放配置

作用：选择要服务化的业务Agent，并配置面向外部系统、内部系统、API、SDK、Webhook 和协作平台的开放渠道。

对应页面：

授权治理

作用：管理谁可以调用这些 Agent 服务，包括应用注册、API Key、权限、租户、配额和调用范围。

对应页面：

调用运行

作用：外部系统、内部系统或协作平台实际调用 Agent 服务，包括创建会话、提交任务、查询状态、获取结果和中止任务。

对应页面：

回调监控

作用：把执行结果、状态变化和异常告警推送回调用方，并监控调用量、失败率、延迟、配额、调用日志和审计记录。

对应页面：

业务Agent

注册 -> 服务 -> 执行 -> 状态 -> 记录 -> 反馈

注册

作用：注册业务Agent，包括 Agent 标识、能力说明、调用协议、所属业务和版本状态。

对应页面：

服务

作用：提供 Agent 服务，接收任务、创建会话、返回结果，并支撑外部调用。

对应页面：

执行

作用：执行业务任务，调用 Tool、Workflow、数据、模型、仿真和设备能力。

对应页面：

状态

作用：暴露运行状态，包括运行中、等待工具、等待确认、失败、完成和中止。

对应页面：

记录

作用：保存运行记录，包括输入、推理过程、工具调用、结果、错误和审计日志。

对应页面：

反馈

作用：回流运行反馈，包括用户评价、失败样例、异常任务和优化建议。

对应页面：

Agent工程

定义 -> 装配 -> 运行 -> 验证 -> 发布 -> 监控优化

定义

作用：定义 Agent 是谁，包括角色、目标、Prompt、模型、输入输出和能力边界。

对应页面：

装配

作用：装配 Agent 能力，包括知识、Ontology、Tool、Workflow、设备能力和凭证。

对应页面：

运行

作用：提供 Runtime，覆盖 Environment、Session、State、Memory、Event 和 Tool Calling。

对应页面：

验证

作用：通过 Test Chat、Debugger、Trace、Eval 和 Simulation Eval 验证 Agent 是否可靠。

对应页面：

发布

作用：管理版本、审批、灰度、回滚、部署和 API 暴露，把工程态 Agent 变成业务Agent。

对应页面：

监控优化

作用：监控运行质量、成本、延迟、失败原因和用户反馈，并回到定义、装配、验证、发布继续迭代。

对应页面：

基础服务

服务发现 -> 服务详情 -> 授权绑定 -> 调用监控

模型服务

作用：管理模型子系统开放给AIP-Agent的模型目录、统一调用、模型路由、Embedding、Vision和多模态服务，不包含模型训练与模型仓库建设。

对应页面：

数据服务

作用：管理数据子系统开放给AIP-Agent的数据目录、Ontology Context、Knowledge Retrieval和Data Tool，不包含数据加工与Ontology建设。

对应页面：

仿真服务

作用：管理仿真子系统开放给AIP-Agent的场景选择、任务推演、Replay、仿真评估和Simulation Tool，不包含仿真场景建设。

对应页面：

设备服务

作用：管理设备子系统开放给AIP-Agent的设备目录、状态遥测、任务下发、Device Tool和动作审计，不包含设备资产建设。
