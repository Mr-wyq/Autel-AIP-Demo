五大基础平台提供数据、模型、部署、仿真、设备能力
Agent 工程定义、装配、验证并发布 Agent
业务 Agent承载已发布 Agent 的运行、状态和记录
生态开放服务把业务 Agent 封装为 API、SDK、Webhook 或协作平台机器人
外部调用方无人机控制软件、内部系统、第三方平台、Slack / GitHub / Lark
回流链路：外部调用日志、回调结果、业务 Agent 状态、Trace、失败样例和用户反馈，会回到 Agent 工程的“监控优化”；如果发现能力不足，再推动五大基础平台补齐数据、模型、部署、仿真或设备能力。

1. 生态开放服务

开放配置 -> 授权治理 -> 调用运行 -> 回调监控

开放配置

作用：选择要对外服务化的业务 Agent，并配置开放渠道，包括 API、SDK、Webhook、无人机控制软件入口和协作平台机器人。

对应页面：

授权治理

作用：管理谁可以调用这些 Agent 服务，包括应用注册、API Key、权限、租户、配额和调用范围。

对应页面：

调用运行

作用：外部系统实际调用 Agent 服务，包括创建会话、提交任务、查询状态、获取结果和中止任务。

对应页面：

回调监控

作用：把执行结果、状态变化和异常告警推送回外部系统，并监控调用量、失败率、延迟、配额、调用日志和审计记录。

对应页面：

注册 -> 服务 -> 执行 -> 状态 -> 记录 -> 反馈

注册

对应页面：

服务

作用：提供 Agent 服务，接收任务、创建会话、返回结果，并支撑外部调用。

记录

反馈

作用：回流运行反馈，包括用户评价、失败样例、异常任务和优化建议。

3. Agent 工程

定义 -> 装配 -> 运行 -> 验证 -> 发布 -> 监控优化

定义

作用：定义 Agent 是谁，包括角色、目标、Prompt、模型、输入输出和能力边界。

对应页面：

装配

作用：装配 Agent 能力，包括知识、Ontology、Tool、Workflow、设备能力和凭证。

对应页面：

运行

作用：提供 Runtime，覆盖 Environment、Session、State、Memory、Event 和 Tool Calling。

对应页面：

验证

作用：通过 Test Chat、Debugger、Trace、Replay、Eval 和仿真测试验证 Agent 是否可靠。

对应页面：

发布

作用：管理版本、审批、灰度、回滚、部署和 API 暴露，把工程态 Agent 变成业务 Agent。

对应页面：

监控优化

作用：监控运行质量、成本、延迟、失败原因和用户反馈，并回到定义、装配、验证、发布继续迭代。

数据 -> 模型 -> 部署 -> 仿真 -> 设备

数据平台

平台自身能力：数据接入、加工、治理、Ontology、知识库和向量索引。
给 AIP 的服务：Data Query、Knowledge Retrieval、Ontology Service、Vector Search、Data Tool。

给 AIP 的服务：Model Catalog、Model Gateway、Model Routing、Embedding、Vision。

给 AIP 的服务：Environment、Runtime Resource、Agent Deployment、Function Deployment。

给 AIP 的服务：Simulation Scenario、Simulation Task、Simulation Eval、Replay、Simulation Tool。
